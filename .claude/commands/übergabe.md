# Übergabebericht erstellen

Erstelle einen strukturierten Übergabebericht für die aktuelle Session.

## Deine Aufgaben:

1. **Zusammenfassung** - Was wurde in dieser Session erarbeitet?
2. **Geänderte Dateien** - Liste alle modifizierten/erstellten Dateien
3. **Git Status** - Prüfe mit `git status` und `git diff --stat`
4. **Offene Punkte** - Was ist noch zu tun?
5. **Nächste Schritte** - Konkrete Aufgaben für den nächsten Chat
6. **Rolle definieren** - Welche Rolle soll der nächste Claude übernehmen?

## Format:

```markdown
# Übergabebericht: [PROJEKTNAME]
**Datum:** [AKTUELLES DATUM]
**Session:** [Kurzbeschreibung was gemacht wurde]

## Erledigt
- [Was wurde fertiggestellt]

## Geänderte Dateien
- `datei.ts` - [Was wurde geändert]

## Offene Punkte
- [ ] [Was noch zu tun ist]

## Nächste Session
**Rolle:** [z.B. Frontend-Entwickler, Bug-Fixer, etc.]
**Aufgabe:** [Konkrete Arbeitsanweisung]
**Erfolg wenn:** [Woran erkennt man erfolgreiche Bearbeitung]
```

## Wichtig:
- Halte es kompakt aber vollständig
- Keine Implementierungsdetails, nur Überblick
- Prüfe CLAUDE.md auf Projektkontext
